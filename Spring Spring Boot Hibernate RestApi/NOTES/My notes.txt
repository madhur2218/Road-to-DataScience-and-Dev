	Dependency Injection: Dependency Injection (DI) is a design pattern used in software development to manage dependencies between objects. It is a form of Inversion of Control (IoC), where the responsibility of creating objects (dependencies) is transferred from a class to an external entity, often a framework or container. This helps in making the code more modular, testable, and flexible.
		-Dependency Injection reduces the need for hardcoded dependencies within classes, enhances modularity, and improves the overall flexibility and testability of your application.
		-Constructor Injection: If a class cannot function without a certain dependency, you should use constructor injection. This makes it clear that the dependency is required and enforces it at the time of object creation.
	-For dependency injection, spring can use autowiring.
	-Spring will look for class that matches class or interface.
	-Spring will inject automatically, hence called autowired.

Qualifiers:
	-@Qualifier annotation is used in Spring to resolve ambiguity when multiple beans of the same type exist.
	
Primary
	-@Primary is an alternative for @Qualifier, but Primary can only have only one for multiple implementations.
	-We cannot mark multiple classes with @Primary.

In Java, @PostConstruct is an annotation used to indicate that a method should be executed after dependency injection is done on a bean but before the bean is put into service.
Key Points:
	Purpose:
	It allows you to perform any necessary initialization tasks, such as setting up default values, connecting to external resources, or performing calculations, after the bean's dependencies have been injected.
	Lifecycle:
	The @PostConstruct method is called only once in the bean's lifecycle, immediately after the bean is created and its dependencies have been injected.
	Usage:
	You can annotate only one method per class with @PostConstruct. The method must not have any parameters and must not return a value.
